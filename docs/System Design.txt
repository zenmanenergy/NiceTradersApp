NICE Traders
Neighborhood International Currency Exchange
Project Definition & Development Plan
1. Overview

NICE Traders (Neighborhood International Currency Exchange) is a mobile platform designed to help people exchange foreign currency in person, safely and conveniently, within their own community.

The core idea:

A traveler returns home with leftover foreign cash (EUR, GBP, JPY, etc.).

Another traveler in the same city is preparing to visit that country and needs that currency.

NICE Traders connects the two people locally.

They meet in a safe public place and exchange cash.

Each person pays a small $1 fee for using the platform.

This eliminates:

Excessive money-exchanger fees

International ATM transaction fees

The inconvenience of returning leftover foreign cash

The difficulty of obtaining small amounts of foreign currency quickly

NICE Traders turns neighborhoods into micro foreign-exchange networks.

2. Problem NICE Traders Solves

Every year, millions of travelers return home with leftover cash from abroad.
That money often sits unused:

Airport kiosks charge 10–25% fees.

Banks offer poor rates or require appointments.

Online services don’t support in-person cash swaps.

Meanwhile, other travelers need cash before they leave:

Airports charge high rates

Banks may not have the desired currency in stock

Traditional currency-ordering takes days

There is currently no low-cost, community-based, peer-to-peer solution for exchanging physical foreign currency.

NICE Traders fills this gap.

3. How NICE Traders Works
Step 1: A user lists foreign currency they want to sell

Choose currency type (EUR, GBP, JPY, etc.)

Enter amount

Set location

Listing appears on the NICE Traders map

Step 2: Another user searches for nearby currency

Search by currency

See nearby sellers on a map (Uber-style)

View listing info (amount, rate, distance)

Step 3: Users pay $1 each to unlock messaging

Protects privacy

Ensures commitment

Removes incentive to avoid fees

Allows chat to negotiate meeting location

Step 4: Users meet in a safe public place

Suggested safe meet-up spots provided

Optional QR confirmation

Exchange cash in person

Step 5: Users rate each other

Builds trust

Flags potential bad actors

Creates a safety layer similar to Uber/Airbnb

6. Technology Strategy (Three-Phase Plan)

The NICE Traders app will be developed in three stages:

PHASE 1 — Browser Prototype
Goal: Build the entire app flow quickly using HTML/CSS/JS

Phase 1 uses a web-based prototype that:

Looks like a mobile app (using a mobile UI CSS framework)

Works inside Chrome DevTools using mobile preview mode

Implements all REST API connections to the backend

Lets us test and refine the UX rapidly

Avoids compiling or installing native apps

Focuses on UI, navigation, data flow, and business logic

Tools & Stack (Phase 1):

HTML + CSS + JS

Framework7 or Ionic CSS (mobile-style UI)

Axios (for API calls)

Chrome dev tools

Python backend with MySQL

Live Server for instant reload

Phase 1 builds:

Entire user flow

All screens & navigation

Listing creation

Search & filters

Messaging UI (without push)

Payments placeholder

Map placeholder

Ratings flow

Local storage auth

Phase 1 does NOT include:

GPS

Push notifications

Native camera access

PayPal native SDK

Background tasks

When the prototype is correct, we move to Phase 2.

PHASE 2 — Native iOS App (SwiftUI)
Goal: Rewrite the prototype into a full native iOS app

Using the Phase 1 prototype as a blueprint:

AI rewrites each HTML/CSS/JS view into SwiftUI view files

Navigation structure is converted into NavigationStack

REST calls become Swift async/await networking

App-wide state becomes Observables / ViewModels

Native iOS-exclusive features added here:

GPS location (CoreLocation)

Native MapKit map

Apple Push Notifications (APNs)

Camera & QR scanning

Apple secure storage (Keychain)

Background refresh

Native payment integration

iOS app lifecycle management

This produces a fully native, App Store-ready iOS app.

PHASE 3 — Native Android App (Jetpack Compose)
Goal: Use AI to port the SwiftUI iOS app into native Android code

After the iOS app is complete:

SwiftUI screens → Jetpack Compose screens

Swift networking → Retrofit/Ktor

Combine/coroutines → Kotlin coroutines + Flow

MapKit → Google Maps SDK

APNs → Firebase Push Notifications

Keychain → EncryptedSharedPreferences

AI handles:

UI conversion

ViewModels

REST API calls

Most navigation

Most state management

Platform-specific UI translations

Developers handle:

Android permissions model

Notification channels

Google Maps setup

Background WorkManager tasks

This produces a fully native Android app.

7. Benefits of the Three-Phase Strategy
Fastest possible prototyping

Web development is extremely fast:

No compile times

No app installs

Instant UI adjustments

Best possible production quality

End result is true native apps, not hybrid.

AI thrives in this pipeline

AI can accurately convert:

Web prototype → SwiftUI

SwiftUI → Jetpack Compose

Scalable architecture

The final apps can grow into large-scale production systems.

Clear separation of concerns

Phase 1 = UI/UX decisions

Phase 2 = iOS feature integration

Phase 3 = Android feature integration

This prevents “design by accident.”
